# Noise

>  시각적 요소(텍스처, 색상, 형태, 밝기 등)에 **무작위적인 변화(Random variation)** 나 불규칙성을 추가하여 현실감과 디테일을 더하는 **절차적(Procedural)** 생성 기법<br> - *위키피디아* -

> 왜 필요한가?<br> 컴퓨터 그래픽은 기본적으로 너무 완벽해 (*완전한 평면* , *완벽한 색* , *반복되는 패턴*) <br> 현실 세계엔 이런 게 거의 없기에<br/> -> 노이즈로 깨줌 <br> - *Chat GPT* -

단순히 원치 않는 잡음(전자 노이즈 등)을 의미하던 것 이 아닌,

컴퓨터 그래픽스에서는 실제 세계의 불규칙한 표면, 질감, 자연 현상을 시뮬레이션하기 위한 핵심 도구로 사용

---

## 노이즈의 역할

**규칙적인 것에 불규칙성을 추가**

- 색상 흔들기

- 표면 거칠기 표현

- 구름, 연기, 불꽃

- 지형의 울퉁불퉁함


## 대표적인 노이즈

| 종류                | 특징         | 용도     |
| ----------------- | ---------- | ------ |
| Random Noise      | 완전 랜덤      | 거의 안 씀 |
| Value Noise       | 보간된 랜덤     | 기본     |
| **Perlin Noise**  | 부드러운 연속성   | 지형, 구름 |
| **Simplex Noise** | Perlin 개선판 | 실시간    |
| FBM (Fractal)     | 노이즈 여러 겹   | 자연물    |

---

## Shader 관점

**Pixel Shader** 에서 `noise(x, y, t)`

`UV` 를 흔들거나 , `Alpha` 를 흔들거나 `Normal`을 변형 하여 사용

또한 **HLSL** 에선, `noise()` 함수를 지원하지 않으므로 

- 주로 `frac(sin(dot(p, float2(x, y))) * z)` 형태의 수학식을 사용하여 의사 난수(Pseudo-random)를 생성

- 내가 사용할 목적으로 `noise` 함수를 작성해서 사용
